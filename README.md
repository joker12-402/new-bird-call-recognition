# 多域特征融合的轻量级鸟类声纹识别
Bird Call Recognition with Multi-Domain Feature Fusion and Lightweight Attention

## 项目概述
面向被动声学监测（PAM）端侧部署需求，提出融合「频域MFCC+时域RMSE+时频域梅尔声谱图能量」的三特征架构，搭配极简通道重标定模块，在保证识别精度的同时提升训练稳定性。

### 核心贡献（对应论文）
1. 多域特征融合：覆盖声信号频谱、能量、时频分布，帧级对齐+标准化保证输入一致性；
2. 轻量级注意力：仅9个参数的通道重标定模块，自适应增强关键特征通道；
3. 训练稳定性：五折交叉验证保证可复现性；
4. 工程适配：纠正数据集泄漏问题，模型参数量4.46M，适配边缘设备部署。

## 数据集说明
### 基础信息
- 数据集名称：Birdsdata 数据集
- 规模：20种鸟类，14311段野外录音样本
- 样本不平衡：最多（凤头麦鸡1195个）vs 最少（灰山鹑29个），不平衡比41.21:1

### 预处理步骤（对应论文 2.1 节）
1. 重采样：统一至 16 kHz；
2. 分帧：帧长128 ms、帧移32 ms，汉明窗抑制频谱泄漏；
3. 特征提取：
   - MFCC：40维，预加重系数0.97，梅尔滤波器组26个；
   - RMSE：帧级时域能量，反映信号强度波动；
   - 梅尔声谱图能量：40维，转换为分贝（dB）尺度增强动态范围；
4. 标准化：仅用训练集估计 μ（均值）和 σ（标准差），验证/测试集复用（修复泄漏关键）；
5. 尺寸统一：双三次插值缩放至 128×128，堆叠为3通道输入张量。

### 划分方式
- 分层随机抽样(旧)：训练集70%、验证集15%、测试集15%（保持类别比例一致）；
- 五折交叉（新）：使用 `StratifiedKFold`，确保每折类别分布均衡。

### 数据集泄漏纠正（详细见 `notes/dataset_leak_fix.md`）
- 原问题：旧实验中标准化时使用了全量数据（训练+验证+测试）的统计量，导致数据泄漏；
- 纠正方案：仅基于训练集计算 μ 和 σ，验证集、测试集严格复用训练集的统计量；
- 影响：纠正后结果更贴近真实场景，避免过拟合导致的性能虚高。

## 实验说明
### 当前运行实验（核心）
- 脚本：`models/train_kfold_multi.py`
- 实验类型：五折交叉消融实验（覆盖所有5个模型）
- 运行状态：已完成4个模型，剩余1个模型待运行（2024-XX-XX更新）
- 核心设置：
  - 优化器：AdamW（lr=1e-3，weight_decay=1e-4）；
  - 损失函数：类别权重修正交叉熵（缓解样本不平衡）；
  - 早停：patience=10，基于验证集加权F1选择最优模型；
  - 正则化：梯度裁剪（max_norm=1.0）、分层Dropout2d。

- 脚本：`models/old_versions/` 下所有文件
- 核心参考：`seed=42` 单种子结果（论文中关键性能数据来源）
- 注意：旧实验未修复数据集泄漏，仅作对比参考，最终以五折交叉结果为准。
